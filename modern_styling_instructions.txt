# Modern Styling Instructions (UI/UX Transformation)

## Design Philosophy: "Digital Glass & Light"
The goal is to create a premium, minimal, and trustworthy interface suitable for a modern Hospital Management System. We will move away from generic admin dashboards to a product that feels "consumer-grade" in polish.

### 1. Color Palette (Modern & Medical)
Avoid high-saturation primary colors. Use calm, sophisticated tones.

*   **Primary Brand**: `Soft Royal Blue` (#3B82F6 -> #2563EB)
    *   Used for: Primary buttons, active states, key data points.
*   **Secondary/Accent**: `Teal / Mint` (#14B8A6)
    *   Used for: Success states, growth indicators, wellness metrics.
*   **Backgrounds**:
    *   **Body**: `#F8FAFC` (Slate 50) - Not pure white, reduces eye strain.
    *   **Cards/Panels**: `#FFFFFF` (White) - With subtle shadow.
    *   **Sidebar/Nav**: `#0F172A` (Slate 900) - High contrast, professional dark mode for navigation relative to content.
*   **Text**:
    *   **Headings**: `#1E293B` (Slate 800)
    *   **Body**: `#475569` (Slate 600)
    *   **Muted**: `#94A3B8` (Slate 400)

### 2. Light & Shadow (Depth)
*   **Shadows**: Do not use hard black shadows. Use colored, diffused shadows.
    *   *CSS*: `box-shadow: 0 4px 20px -2px rgba(59, 130, 246, 0.1);` (Blue-tinted glow).
*   **Borders**: Minimize mismatched borders. Use standard 1px borders in very light gray (`#E2E8F0`) only when necessary. Prefer spacing and background separation over lines.

### 3. Typography
*   **Font Family**: `Inter` (Google Font) or `Plus Jakarta Sans`.
*   **Weights**:
    *   Bold (600/700) for big numbers and page titles.
    *   Medium (500) for navigation and buttons.
    *   Regular (400) for data tables and body text.
*   **Spacing**: Increase `line-height` (leading) to 1.6 for lists and paragraphs.

### 4. Component Styling Guidelines

#### **Buttons**
*   **Primary**: Rounded corners (not pill, not square -> `rounded-lg`). Gradient background from Blue-500 to Blue-600.
*   **Secondary**: White background, Border Slate-200, Text Slate-700. Hover: Slate-50.
*   **Ghost**: Transparent background, text only. Used for table actions.

#### **Forms & Inputs**
*   **Style**: minimal borders.
*   **State**: On focus, the border should not just turn blue, but have a "ring" focus ring (`ring-2 ring-blue-100`).
*   **Labels**: Small, uppercase, tracking-wide (`text-xs font-semibold tracking-wider text-slate-500`).

#### **Data Tables (Critical for HMS)**
*   **Header**: Light gray background (`bg-slate-50`), text uppercase and smaller.
*   **Rows**: White background. Hover effect (`hover:bg-slate-50`) is mandatory.
*   **Density**: Comfortable padding (`py-4`). Don't cram data. Use distinct avatars for users/patients.

#### **Cards / Widgets**
*   **Glassmorphism (Subtle)**: For overlay elements (modals, floating menus), use a backdrop blur:
    *   `bg-white/90 backdrop-blur-md border border-white/20 shadow-xl`
*   **Roundedness**: `rounded-xl` or `rounded-2xl` for container cards.

### 5. Layout & Navigation
*   **Sidebar**: Collapsible. Icons should be outlined (e.g., Lucide-React or Heroicons Outline) for a clean look.
*   **Top Bar**: Minimal. Breadcrumbs for navigation. Universal "Search" bar in the center.

### 6. Micro-Interactions
*   **Transitions**: All hover states must have `transition-all duration-200 ease-in-out`.
*   **Feedback**: Clicking a button should have a scale effect (`active:scale-95`).
